# Bilibili 탄막 리스너 - 빠른 시작

몇 단계로 Bilibili 라이브룸 탄막을 실시간으로 모니터링하고, AI 응답, 백그라운드 LLM, Bilibili 읽기/쓰기 도구와 함께 사용할 수 있습니다.

---

## 1. 연결 상태

상단 카드는 플러그인 실행 상태를 실시간으로 표시합니다:

- **상태 표시등** — 회색=연결 안 됨, 초록=연결됨, 노랑=연결 중, 빨강=오류
- **수신 수** — 수신한 탄막 / 선물 / SC 누적 건수
- **필터됨** — 필터 규칙에 의해 차단된 메시지 건수
- **버퍼** — 집계 푸시를 대기 중인 탄막 건수
- **인기** — 현재 라이브룸의 인기 값

---

## 2. Bilibili 계정 로그인

「Bilibili 계정」 영역을 클릭하여 로그인 패널을 펼칩니다:

- **QR 로그인 (권장)**: 「QR 로그인」 클릭 → Bilibili App 으로 QR 스캔 → 자동 확인 대기
- **인증 정보 확인**: 현재 로그인 상태, 사용자 이름, UID, 만료일 확인
- **인증 정보 재로드**: 로그인 상태를 수동으로 새로 고침
- **로그아웃**: 로컬 암호화 인증 정보를 삭제하고 로그아웃

> 민감한 정보 유출을 방지하기 위해 Cookie 수동 입력 항목은 제거되었습니다. 게스트 모드에서는 탄막 수신만 가능하며, 탄막 전송이나 고급 필터링은 사용할 수 없습니다.

---

## 3. 라이브룸 설정

「라이브룸 설정」 영역에서:

1. **라이브룸 ID 입력** — 라이브룸 URL에서 가져옵니다. 예: `https://live.bilibili.com/22925943` 의 `22925943`
2. **「방 전환」 클릭** — 새 방 ID를 적용
3. **「모니터링 시작」 클릭** — 탄막 서버에 연결하여 탄막 수신 시작

**라이브 탄막 전송**:

- 「NEKO 가 말하게 하기」 끔: 입력 내용을 라이브룸으로 바로 전송
- 「NEKO 가 말하게 하기」 켬: 입력 내용 + 라이브룸 컨텍스트를 NEKO에 전달하여, 캐릭터 설정에 따라 생성된 응답을 전송

---

## 4. AI 푸시 설정

탄막을 AI에 어떻게 푸시할지 제어합니다:

- **푸시 간격(초)** — 탄막을 집계하여 AI에 푸시하는 시간 간격. 10~30 초 권장. 너무 짧으면 AI 응답이 잦고, 너무 길면 응답 지연이 커집니다
- **탄막 최대 길이** — Bilibili는 20자 제한, AI 응답이 초과되면 자동으로 잘립니다. 20을 유지하는 것을 권장
- **대상 AI 이름** — 어떤 AI가 탄막 푸시를 받을지 지정. 비워두면 기본 AI로 푸시
- **주인의 Bilibili UID / 사용자 이름** — 주인 계정을 설정하면 NEKO가 주인 메시지를 특별 처리 (우선 응답, 다른 어조 등)

---

## 5. 실시간 탄막 스트림

수신한 탄막, 선물, SC를 실시간으로 표시:

- **탄막 (분홍 왼쪽 테두리)** — 일반 사용자 탄막. 사용자 이름, 레벨, 팬 배지 표시
- **선물 (금색 왼쪽 테두리)** — 사용자가 선물을 보낸 기록
- **SC / Super Chat (초록 왼쪽 테두리)** — 유료 강조 메시지

**제어 버튼**:

- **자동 스크롤** — 켜면 새 탄막이 자동으로 보이는 영역으로 스크롤됩니다
- **일시정지 / 계속** — 탄막 스트림 표시 갱신을 일시 정지하거나 재개
- **비우기** — 현재 탄막 표시 기록 비우기

---

## 6. Bilibili 읽기 도구

Bilibili 공개 데이터를 읽으며, 쓰기 권한이 필요 없어 안전하게 호출할 수 있습니다. 위쪽 「키워드 / BV 번호 / UID / 즐겨찾기 ID」 등 필드를 입력한 뒤 해당 버튼 클릭:

- **영상 검색** — 키워드로 영상 검색. 필수: 키워드
- **인기 영상** — 사이트 전체 인기 영상 목록. 필수 파라미터 없음
- **인기 검색어** — Bilibili 실시간 인기 검색 순위. 필수 파라미터 없음
- **주간 필수 시청** — 주간 필수 시청 추천. 필수 파라미터 없음
- **랭킹** — 지정 카테고리 랭킹. 필수: 정렬/카테고리 (`all` / `game` / `dance` 등)
- **영상 정보** — 영상 상세 정보 가져오기. 필수: BV 번호
- **영상 댓글** — 영상 댓글 목록 가져오기. 필수: BV 번호
- **영상 자막** — AI 자막 내용 가져오기. 필수: BV 번호
- **과거 탄막** — 영상의 과거 탄막 가져오기. 필수: BV 번호
- **사용자 정보** — 사용자 프로필 가져오기. 필수: UID
- **사용자 업로드** — 사용자가 업로드한 영상 목록 가져오기. 필수: UID
- **즐겨찾기 목록** — 사용자 즐겨찾기 목록 가져오기. 필수: UID
- **즐겨찾기 내용** — 즐겨찾기 폴더의 영상 목록 가져오기. 필수: 즐겨찾기 media_id

호출 결과는 모두 「Bilibili 도구 결과」 영역에 표시됩니다.

---

## 7. Bilibili 쓰기 도구

Bilibili에 쓰기 작업을 수행합니다. **계정에 영향을 미치므로** 신중하게 사용하세요:

- **댓글/답글 작성** — 영상에 댓글을 달거나 댓글에 답글. 필수: BV 번호 + 댓글 내용; 답글은 댓글 rpid도 필요
- **동태 게시** — 새 동태 게시. 필수: 동태 텍스트 내용 (이미지 첨부 가능)
- **쪽지 보내기** — 지정 사용자에게 쪽지 발송. 필수: 상대 UID + 쪽지 내용

- 「NEKO 가 말하게 하기」를 켜면 댓글/동태/쪽지를 먼저 NEKO가 캐릭터 설정에 따라 생성한 뒤 전송합니다
- 쓰기 도구 버튼은 빨간색입니다. 호출 전 계정 로그인 여부, 내용 정확성, 대상 정확성을 반드시 확인하세요

---

## 8. 백그라운드 LLM 설정

활성화하면 탄막이 집계된 후 지정 LLM으로 전달되어 가이드 문구가 생성되며, NEKO가 라이브룸 분위기에 더 자연스럽게 반응할 수 있습니다.

**기본 설정**:

- **활성화 스위치** — 백그라운드 LLM 기능 켜기/끄기
- **API 주소** — OpenAI 호환 엔드포인트. 예: `https://api.openai.com/v1/chat/completions`
- **모델 이름** — 예: `gpt-4o-mini`, `deepseek-chat`
- **API Key** — API 키 (입력 시 기본 숨김, 클릭하여 보기)
- **집계 윈도우** — LLM 요약을 트리거하기 전 수집할 탄막 수, 10~20 권장
- **최대 샘플 수** — 탄막 샘플 풀의 최대 용량. 초과 시 시간순으로 오래된 탄막이 제거됩니다

**고급 설정** (「고급 설정」 클릭하여 펼치기):

- **캣걸 이름** — 프롬프트의 `{name}` 자리표시자가 자동 치환됨
- **지식 베이스 컨텍스트** — 캐릭터 페르소나, 말버릇, 흔한 밈. `{name}` 자리표시자 지원
- **사용자 프로필 요약** — 스트리머/사용자 기본 프로필. LLM 참고용
- **프롬프트 템플릿** — 사용자 정의 System Prompt. `{name}`과 `{knowledge_context}` 자리표시자 지원. 비워두면 기본 템플릿 사용

> 설정을 마친 후 「설정 저장」을 클릭한 뒤 활성화 스위치를 켜세요. 「테스트」를 클릭하면 API 연결을 확인할 수 있습니다.

## 자주 묻는 질문

**QR 로그인 실패?** App이 로그인되어 있는지 확인. QR 코드는 2분 동안 유효하므로 새로 고침 후 다시 스캔하세요.

**탄막 리스너가 반응 없음?** 방 ID가 올바른지, 네트워크가 정상인지, 계정이 로그인되었는지 확인.

**AI가 탄막에 응답하지 않음?** 푸시 간격이 설정되었는지, 백그라운드 LLM이 활성화되었는지, API 설정이 올바른지 확인.

**탄막 전송 실패?** 로그인 여부와 해당 라이브룸의 탄막 전송 권한을 확인 (일부 방은 계정 레벨 제한이 있음).

**API 호출 오류?** API 주소, 모델 이름, API Key가 올바른지 확인. 「테스트」를 클릭해 원인을 진단하세요.
