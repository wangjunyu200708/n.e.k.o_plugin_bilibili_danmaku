# B 站彈幕監聽 - 快速開始

幾步設定即可即時監聽 B 站直播間彈幕，搭配 AI 回覆、背景 LLM、B 站讀寫工具。

---

## 一、連線狀態

頂部卡片即時顯示外掛運行狀態：

- **狀態燈** — 灰色=未連線，綠色=已連線，黃色=連線中，紅色=錯誤
- **已收到** — 累計收到的彈幕 / 禮物 / SC 總條數
- **已過濾** — 被過濾規則攔截的訊息條數
- **緩衝** — 正在等待聚合推播的彈幕條數
- **人氣** — 目前直播間人氣值

---

## 二、登入 B 站帳號

點擊「B 站帳號」區域展開登入面板：

- **掃碼登入**（推薦）：點擊「掃碼登入」→ 用 B 站 App 掃描 QR Code → 等待自動確認
- **檢查憑證**：查看目前登入狀態、使用者名稱、UID、有效期限
- **重新載入憑證**：手動重新整理登入狀態
- **清除登入**：刪除本地加密憑證，登出帳號

> 不再提供 Cookie 手動填寫入口，避免敏感資訊外洩。訪客模式可接收彈幕，但無法傳送彈幕或使用進階過濾。

---

## 三、直播間設定

在「直播間設定」區域：

1. **輸入直播間 ID** — 從直播間 URL 取得，例如 `https://live.bilibili.com/22925943` 中的 `22925943`
2. **點擊「切換房間」** — 讓新房間 ID 生效
3. **點擊「開始監聽」** — 連線彈幕伺服器，開始接收彈幕

**傳送直播彈幕**：

- 關閉「是否讓 NEKO 說」：直接將輸入框內容傳送至直播間
- 開啟「是否讓 NEKO 說」：將輸入內容 + 直播間上下文交給 NEKO，由 NEKO 依人設產生回覆後再傳送

---

## 四、AI 推播設定

控制彈幕如何推播給 AI 處理：

- **推播間隔（秒）** — 彈幕聚合後推播給 AI 的時間間隔，建議 10~30 秒。太短 AI 回應頻繁，太長回覆延遲大
- **彈幕最大長度** — B 站限制 20 字元，AI 產生的回覆超出部分會被自動截斷，建議保持 20
- **目標 AI 名稱** — 指定哪個 AI 接收彈幕推播，留空則推播給預設 AI
- **主人 B 站 UID / 使用者名稱** — 設定主人帳號後，NEKO 會對主人訊息特殊處理（優先回覆、不同語氣等）

---

## 五、即時彈幕流

即時顯示收到的彈幕、禮物和 SC：

- **彈幕（粉色）** — 一般使用者彈幕，顯示使用者名稱、等級、粉絲牌
- **禮物（金色）** — 使用者贈送禮物的紀錄
- **SC（Super Chat）（綠色）** — 付費醒目留言

**控制按鈕**：

- **自動捲動**：開啟後新彈幕會自動捲動到可見區域
- **暫停 / 繼續**：暫停或恢復彈幕流的顯示重新整理
- **清空**：清空目前彈幕顯示紀錄

---

## 六、B 站讀工具

讀取 B 站公開資料，無需寫入權限，可安全呼叫。填寫上方「關鍵字 / BV 號 / UID / 收藏夾 ID」等欄位，再點擊對應按鈕：

- **搜尋影片** — 依關鍵字搜尋影片。必填：關鍵字
- **熱門影片** — 全站熱門影片清單
- **熱搜詞** — B 站即時熱搜榜單
- **每週必看** — 每週必看精選
- **排行榜** — 指定分區排行榜。必填：排序/分區（如 `all`/`game`/`dance`）
- **影片資訊** — 取得影片詳情。必填：BV 號
- **影片留言** — 取得影片留言清單。必填：BV 號
- **影片字幕** — 取得 AI 字幕內容。必填：BV 號
- **歷史彈幕** — 取得影片歷史彈幕。必填：BV 號
- **使用者資訊** — 取得使用者資料。必填：UID
- **使用者投稿** — 取得使用者投稿影片清單。必填：UID
- **收藏夾清單** — 取得使用者收藏夾清單。必填：UID
- **收藏夾內容** — 取得收藏夾內影片清單。必填：收藏夾 media_id

呼叫結果統一顯示在「B 站工具結果」區域。

---

## 七、B 站寫工具

對 B 站執行寫入操作，**會對帳號產生影響**，請謹慎使用：

- **發留言/回覆** — 在影片下發留言或回覆留言。必填：BV 號 + 留言內容；回覆需額外填寫留言 rpid
- **發動態** — 發布新動態。必填：動態文字內容（可附圖片）
- **發私訊** — 向指定使用者發私訊。必填：對方 UID + 私訊內容

- **是否讓 NEKO 說**：開啟後，留言/動態/私訊會先交給 NEKO 依人設產生內容，再傳送
- 寫工具按鈕為紅色，呼叫前請確認：帳號已登入、內容無誤、對象正確

---

## 八、背景 LLM 設定

啟用後，彈幕會被聚合後送給指定 LLM 產生引導詞，讓 NEKO 更自然地回應直播間氛圍。

**基礎設定**：

- **啟用開關** — 開啟/關閉背景 LLM 功能
- **API 位址** — 相容 OpenAI 格式的端點，例如 `https://api.openai.com/v1/chat/completions`
- **模型名稱** — 例如 `gpt-4o-mini`、`deepseek-chat`
- **API Key** — API 金鑰（輸入時預設隱藏，點擊即可檢視）
- **聚合視窗** — 收集多少條彈幕後觸發一次 LLM 總結，建議 10~20
- **最大取樣數** — 彈幕取樣池最大容量，超出後依時間淘汰舊彈幕

**進階設定**（點擊「進階設定」展開）：

- **貓娘名字** — 用於提示詞中的 `{name}` 佔位符自動替換
- **知識庫上下文** — 角色人設、口頭禪、常見梗，支援 `{name}` 佔位符
- **使用者畫像摘要** — 主播/使用者基本畫像，供 LLM 參考
- **提示詞範本** — 自訂 System Prompt，支援 `{name}` 與 `{knowledge_context}` 佔位符，留空則使用預設範本

> 設定完成後點擊「儲存設定」，再開啟啟用開關。點擊「測試」可驗證 API 連通性。

## 常見問題

**掃碼登入失敗？** 確認 App 已登入，QR Code 2 分鐘內有效，重新整理後再掃描一次
**彈幕監聽無反應？** 檢查房間 ID 是否正確、網路是否正常、帳號是否已登入
**AI 不回覆彈幕？** 確認推播間隔已設定、背景 LLM 已啟用且 API 設定正確
**傳送彈幕失敗？** 確認已登入且直播間有發彈幕權限（部分房間限制帳號等級）
**API 呼叫錯誤？** 檢查 API 位址、模型名稱、API Key 是否正確，點擊「測試」排查
