# Прослушивание данму Bilibili — быстрый старт

Несколько шагов настройки, чтобы в реальном времени отслеживать данму комнаты трансляции Bilibili в связке с AI-ответами, фоновой LLM и инструментами чтения/записи Bilibili.

---

## 1. Состояние подключения

Верхняя карточка в реальном времени показывает состояние работы плагина:

- **Индикатор статуса** — Серый = не подключено, зелёный = подключено, жёлтый = подключение, красный = ошибка
- **Получено** — Накопительное количество полученных данму / подарков / SC
- **Отфильтровано** — Количество сообщений, отброшенных правилами фильтра
- **Буфер** — Количество данму, ожидающих агрегированной отправки
- **Популярность** — Текущий показатель популярности комнаты трансляции

---

## 2. Вход в аккаунт Bilibili

Кликните область «Аккаунт Bilibili», чтобы развернуть панель входа:

- **Вход по QR-коду** (рекомендуется): нажмите «Вход по QR-коду» → отсканируйте QR-код приложением Bilibili App → дождитесь автоматического подтверждения
- **Проверка учётных данных**: посмотрите текущее состояние входа, имя пользователя, UID и срок действия
- **Перезагрузка учётных данных**: вручную обновите состояние входа
- **Очистить вход**: удалить локально зашифрованные учётные данные и выйти из аккаунта

> Поле для ручного ввода Cookie удалено, чтобы не допустить утечки конфиденциальных данных. Гостевой режим позволяет получать данму, но не отправлять их и не использовать продвинутую фильтрацию.

---

## 3. Настройка комнаты трансляции

В разделе «Настройки комнаты трансляции»:

1. **Введите ID комнаты** — возьмите его из URL комнаты трансляции, например `22925943` из `https://live.bilibili.com/22925943`
2. **Нажмите «Сменить комнату»** — чтобы применить новый ID комнаты
3. **Нажмите «Начать прослушивание»** — подключение к серверу данму и приём сообщений

**Отправка данму в эфир**:

- Если «Пусть NEKO говорит» выключено: содержимое поля ввода отправляется в комнату напрямую
- Если «Пусть NEKO говорит» включено: ввод и контекст комнаты трансляции передаются NEKO, которая по своему персонажу формирует ответ перед отправкой

---

## 4. Настройки отправки в AI

Управляет тем, как данму передаются AI на обработку:

- **Интервал отправки (сек)** — Интервал, через который агрегированные данму отправляются AI. Рекомендуется 10–30 секунд. Слишком короткий — AI реагирует слишком часто, слишком длинный — большая задержка ответа
- **Максимальная длина данму** — Bilibili ограничивает данму 20 символами; ответ AI сверх лимита автоматически обрезается. Рекомендуется оставить 20
- **Имя целевого AI** — Указывает, какой AI получает поток данму. Если оставить пустым, отправка пойдёт в AI по умолчанию
- **UID / имя владельца на Bilibili** — После настройки аккаунта владельца NEKO будет особо обрабатывать его сообщения (приоритетные ответы, другая интонация и т. п.)

---

## 5. Поток данму в реальном времени

В реальном времени показывает полученные данму, подарки и SC:

- **Данму (Розовый)** — Обычные пользовательские данму, отображаются имя пользователя, уровень и фан-бейдж
- **Подарок (Золотой)** — Записи об отправленных пользователями подарках
- **SC (Super Chat) (Зелёный)** — Платное выделенное сообщение

**Кнопки управления**:

- **Автопрокрутка**: при включении новые данму автоматически прокручиваются в видимую область
- **Пауза / Продолжить**: приостановить или возобновить обновление потока данму
- **Очистить**: очистить текущую историю отображения данму

---

## 6. Инструменты чтения Bilibili

Чтение публичных данных Bilibili без прав на запись — безопасно вызывать. Заполните поля «ключевое слово / BV / UID / ID избранного» сверху и нажмите соответствующую кнопку:

- **Поиск видео** — Поиск видео по ключевому слову. Обязательно: Ключевое слово
- **Популярные видео** — Список популярных видео по всему сайту
- **Горячий поиск** — Рейтинг популярных запросов Bilibili в реальном времени
- **Рекомендации недели** — Подборка «Рекомендации недели»
- **Рейтинг** — Рейтинг указанной категории. Обязательно: Сортировка/категория (`all`/`game`/`dance` и т. п.)
- **Информация о видео** — Получить подробности о видео. Обязательно: BV
- **Комментарии к видео** — Получить список комментариев к видео. Обязательно: BV
- **Субтитры видео** — Получить AI-субтитры. Обязательно: BV
- **История данму** — Получить исторические данму видео. Обязательно: BV
- **Информация о пользователе** — Получить профиль пользователя. Обязательно: UID
- **Загрузки пользователя** — Получить список видео, загруженных пользователем. Обязательно: UID
- **Список избранного** — Получить список папок избранного пользователя. Обязательно: UID
- **Содержимое избранного** — Получить список видео в папке избранного. Обязательно: media_id избранного

Результаты вызова отображаются в области «Результаты инструментов Bilibili».

---

## 7. Инструменты записи Bilibili

Выполняют операции записи в Bilibili — **влияют на ваш аккаунт**, используйте с осторожностью:

- **Комментарий/ответ** — Оставить комментарий под видео или ответить на комментарий. Обязательно: BV + текст комментария; для ответа нужен также rpid комментария
- **Опубликовать пост** — Опубликовать новый пост (динамику). Обязательно: Текст поста (можно с изображениями)
- **Личное сообщение** — Отправить личное сообщение пользователю. Обязательно: UID получателя + текст сообщения

- **Пусть NEKO говорит**: при включении комментарий / пост / ЛС сначала формирует NEKO согласно своему персонажу, и только затем сообщение отправляется
- Кнопки инструментов записи красные. Перед вызовом убедитесь: аккаунт авторизован, содержимое корректно, цель указана верно

## 8. Настройки фоновой LLM

После включения данму агрегируются и отправляются в указанную LLM для генерации направляющей подсказки, чтобы NEKO более естественно отвечала атмосфере комнаты трансляции.

**Базовая настройка**:

- **Переключатель** — Включить/выключить функцию фоновой LLM
- **API-адрес** — Совместимая с OpenAI конечная точка, например `https://api.openai.com/v1/chat/completions`
- **Имя модели** — Например, `gpt-4o-mini`, `deepseek-chat`
- **API Key** — API-ключ (по умолчанию скрыт при вводе, нажмите для показа)
- **Окно агрегации** — Сколько данму собирать перед запуском суммирования LLM, рекомендуется 10–20
- **Макс. размер выборки** — Максимальная вместимость пула данму; при превышении старые данму удаляются по времени

**Расширенные настройки** (нажмите «Расширенные настройки», чтобы развернуть):

- **Имя котодевочки** — Подставляется в плейсхолдер `{name}` подсказок
- **Контекст базы знаний** — Персонаж, любимые фразы, распространённые мемы; поддерживает плейсхолдер `{name}`
- **Сводка профиля пользователя** — Базовое описание стримера/пользователей для LLM
- **Шаблон подсказки** — Пользовательский System Prompt; поддерживает плейсхолдеры `{name}` и `{knowledge_context}`. Если оставить пустым, используется шаблон по умолчанию

> После настройки нажмите «Сохранить конфигурацию», затем включите переключатель. Кнопка «Тест» проверяет доступность API.

## Часто задаваемые вопросы

**Не удаётся войти по QR-коду?** Убедитесь, что в App выполнен вход; QR-код действителен 2 минуты — обновите его и отсканируйте снова
**Прослушивание данму не реагирует?** Проверьте корректность ID комнаты, состояние сети и факт авторизации в аккаунте
**AI не отвечает на данму?** Убедитесь, что задан интервал отправки, фоновая LLM включена и настройки API верны
**Не удаётся отправить данму?** Убедитесь, что вы авторизованы и в этой комнате трансляции есть право отправлять данму (некоторые комнаты ограничивают по уровню аккаунта)
**Ошибка вызова API?** Проверьте API-адрес, имя модели и API Key; нажмите «Тест» для диагностики
