# Bilibili 弾幕モニター - クイックスタート

数ステップで Bilibili 配信ルームの弾幕をリアルタイム監視し、AI 返信、バックグラウンド LLM、Bilibili 読み書きツールと連携できます。

---

## 1. 接続ステータス

上部カードがプラグインの稼働状態をリアルタイムに表示します：

- **ステータスランプ** — 灰=未接続、緑=接続済み、黄=接続中、赤=エラー
- **受信数** — 受信した弾幕 / ギフト / SC の累計件数
- **フィルタ済み** — フィルタルールでブロックされたメッセージ件数
- **バッファ** — 集約プッシュ待ちの弾幕件数
- **人気度** — 現在の配信ルームの人気度

---

## 2. Bilibili アカウントへのログイン

「Bilibili アカウント」エリアをクリックしてログインパネルを展開：

- **QR コードログイン（推奨）**：「QR コードログイン」をクリック → Bilibili App で QR コードをスキャン → 自動確認を待つ
- **資格情報チェック**：現在のログイン状態、ユーザー名、UID、有効期限を確認
- **資格情報の再読み込み**：ログイン状態を手動で更新
- **ログイン解除**：ローカル暗号化資格情報を削除しログアウト

> 機密情報の漏洩を防ぐため、Cookie 手動入力欄は廃止されました。ゲストモードでは弾幕の受信は可能ですが、弾幕送信や高度なフィルタリングは利用できません。

---

## 3. 配信ルームの設定

「配信ルーム設定」エリアで：

1. **ルーム ID を入力** — 配信ルーム URL から取得します。例：`https://live.bilibili.com/22925943` の `22925943`
2. **「ルーム切替」をクリック** — 新しいルーム ID を反映
3. **「監視開始」をクリック** — 弾幕サーバーに接続し、弾幕の受信を開始

**ライブ弾幕の送信**：

- 「NEKO に話させる」をオフ：入力内容をそのまま配信ルームに送信
- 「NEKO に話させる」をオン：入力内容と配信ルームのコンテキストを NEKO に渡し、人格設定に基づいて生成された返信を送信

---

## 4. AI プッシュ設定

弾幕を AI に渡す方法を制御します：

- **プッシュ間隔（秒）** — 弾幕を集約して AI にプッシュする時間間隔。10〜30 秒を推奨。短すぎると AI の反応が頻発し、長すぎると返信遅延が大きくなります
- **弾幕最大長** — Bilibili の上限は 20 文字。AI の返信が超過した分は自動で切り詰められます。20 のままを推奨
- **対象 AI 名** — どの AI が弾幕プッシュを受け取るかを指定。空欄でデフォルト AI に送信
- **主人の Bilibili UID / ユーザー名** — 主人アカウントを設定すると、NEKO は主人のメッセージを特別扱い（優先返信、口調変更など）します

---

## 5. リアルタイム弾幕ストリーム

受信した弾幕、ギフト、SC をリアルタイム表示：

- **弾幕（ピンクの左ボーダー）** — 一般ユーザーの弾幕。ユーザー名、レベル、ファンバッジを表示
- **ギフト（ゴールドの左ボーダー）** — ユーザーがギフトを贈った記録
- **SC / Super Chat（グリーンの左ボーダー）** — 有料の目立つメッセージ

**コントロールボタン**：

- **自動スクロール** — オンにすると新着弾幕が自動でスクロールして表示されます
- **一時停止 / 再開** — 弾幕ストリームの表示更新を一時停止／再開
- **クリア** — 現在の弾幕表示履歴をクリア

---

## 6. Bilibili 読み取りツール

Bilibili の公開データを取得します。書き込み権限不要で安全に呼び出せます。上部の「キーワード / BV 番号 / UID / お気に入り ID」などのフィールドを記入し、対応するボタンをクリック：

- **動画検索** — キーワードで動画を検索。必須：キーワード
- **人気動画** — サイト全体の人気動画リスト。必須パラメータなし
- **ホット検索** — Bilibili のリアルタイムホット検索ランキング。必須パラメータなし
- **週刊おすすめ** — 週刊おすすめピックアップ。必須パラメータなし
- **ランキング** — 指定カテゴリのランキング。必須：並び順/カテゴリ（`all` / `game` / `dance` など）
- **動画情報** — 動画詳細を取得。必須：BV 番号
- **動画コメント** — 動画のコメント一覧を取得。必須：BV 番号
- **動画字幕** — AI 字幕の内容を取得。必須：BV 番号
- **過去弾幕** — 動画の過去弾幕を取得。必須：BV 番号
- **ユーザー情報** — ユーザープロフィールを取得。必須：UID
- **ユーザー投稿** — ユーザーの投稿動画一覧を取得。必須：UID
- **お気に入りリスト** — ユーザーのお気に入り一覧を取得。必須：UID
- **お気に入り内容** — お気に入り内の動画一覧を取得。必須：お気に入りの media_id

呼び出し結果はすべて「Bilibili ツール結果」エリアに表示されます。

---

## 7. Bilibili 書き込みツール

Bilibili に対する書き込み操作を実行します。**アカウントに影響します**ので慎重にご利用ください：

- **コメント／返信投稿** — 動画にコメントしたり、コメントに返信。必須：BV 番号 + コメント内容；返信時はコメント rpid も必要
- **ダイナミック投稿** — 新しいダイナミックを公開。必須：ダイナミックのテキスト（画像添付可）
- **DM 送信** — 指定ユーザーに DM を送信。必須：相手の UID + メッセージ内容

- 「NEKO に話させる」をオンにすると、コメント／ダイナミック／DM をまず NEKO が人格設定に基づいて生成してから送信します
- 書き込みツールのボタンは赤色です。呼び出し前にアカウントログイン済み・内容に誤りがない・対象が正しい、を必ず確認してください

---

## 8. バックグラウンド LLM 設定

有効化すると、弾幕が集約された後に指定 LLM へ送られ、ガイダンスプロンプトが生成されます。NEKO が配信ルームの雰囲気により自然に応答できるようになります。

**基本設定**：

- **有効スイッチ** — バックグラウンド LLM 機能のオン/オフ
- **API アドレス** — OpenAI 互換のエンドポイント。例：`https://api.openai.com/v1/chat/completions`
- **モデル名** — 例：`gpt-4o-mini`、`deepseek-chat`
- **API Key** — API キー（入力時はデフォルトで非表示、クリックで表示）
- **集約ウィンドウ** — LLM 要約をトリガーするまでに集める弾幕数。10〜20 を推奨
- **最大サンプル数** — 弾幕サンプルプールの最大容量。超過時は時間順に古い弾幕が破棄されます

**詳細設定**（「詳細設定」をクリックで展開）：

- **猫娘の名前** — プロンプト内の `{name}` プレースホルダーを自動置換
- **知識ベースコンテキスト** — キャラクター人格、口癖、よくあるネタ。`{name}` プレースホルダーに対応
- **ユーザープロファイル要約** — 配信者／ユーザーの基本プロファイル。LLM が参照
- **プロンプトテンプレート** — カスタム System Prompt。`{name}` と `{knowledge_context}` プレースホルダーに対応。空欄でデフォルトテンプレートを使用

> 設定後は「設定を保存」をクリックし、有効スイッチをオンにしてください。「テスト」をクリックすると API 接続性を検証できます。

## よくある質問

**QR コードログインが失敗する？** App がログイン済みか確認。QR コードは 2 分間有効。更新して再スキャンしてください。

**弾幕監視が反応しない？** ルーム ID が正しいか、ネットワークが正常か、アカウントがログイン済みかを確認。

**AI が弾幕に返信しない？** プッシュ間隔の設定、バックグラウンド LLM の有効化、API 設定が正しいかを確認。

**弾幕送信が失敗する？** ログイン済みで、その配信ルームで弾幕送信権限があることを確認（一部ルームはアカウントレベル制限あり）。

**API 呼び出しエラー？** API アドレス、モデル名、API Key が正しいかを確認。「テスト」で原因を切り分けてください。
