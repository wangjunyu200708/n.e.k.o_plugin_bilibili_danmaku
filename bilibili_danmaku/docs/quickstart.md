# B站弹幕监听 - 快速开始

几步设置实时 B站直播间弹幕监听，搭配 AI 回复、背景 LLM、B站读写工具。


## 一、连接状态

顶部卡片实时显示插件运行状态：

- **状态灯** — 灰色=未连接，绿色=已连接，黄色=连接中，红色=错误
- **已收到** — 累计收到的弹幕 / 礼物 / SC 总条数
- **已过滤** — 被过滤规则拦截的消息条数
- **缓冲** — 正在等待聚合推送的弹幕条数
- **人气** — 当前直播间人气值


## 二、登录 B站账号

点击「B站账号」区域展开登录面板：

- **扫码登录（推荐）**：点击「扫码登录」→ 用 B站 App 扫描二维码 → 等待自动确认
- **检查凭据**：查看当前登录状态、用户名、UID、有效期
- **重载凭据**：手动刷新登录状态
- **清除登录**：删除本地加密凭据，退出登录

> 不再提供 Cookie 手动填写入口，避免敏感信息泄露。游客模式可接收弹幕，但无法发送弹幕或使用高级过滤。


## 三、直播间设置

在「直播间设置」区域：

1. **输入直播间 ID** — 从直播间 URL 获取，如 `https://live.bilibili.com/22925943` 中的 `22925943`
2. **点击「切换房间」** — 使新房间 ID 生效
3. **点击「▶ 开始监听」** — 连接弹幕服务器，开始接收弹幕

**发送直播弹幕**：

- 关闭「是否让NEKO说」：直接发送输入框内容到直播间
- 开启「是否让NEKO说」：把输入内容 + 直播间上下文交给 NEKO，由 NEKO 按人设生成回复后再发送


## 四、AI 推送设置

控制弹幕如何推送给 AI 处理：

- **推送间隔（秒）** — 弹幕聚合后推送给 AI 的时间间隔，建议 10~30 秒。太短 AI 响应频繁，太长回复延迟大
- **弹幕最大长度** — B站限制 20 字符，AI 生成的回复超出部分会被自动截断，建议保持 20
- **目标 AI 名称** — 指定哪个 AI 接收弹幕推送，留空则推送给默认 AI
- **主人 B站 UID / 用户名** — 设置主人账号后，NEKO 会对主人消息特殊处理（优先回复、不同语气等）


## 五、实时弹幕流

实时显示收到的弹幕、礼物和 SC：

- **弹幕（粉色左边框）** — 普通用户弹幕，显示用户名、等级、粉丝牌
- **礼物（金色左边框）** — 用户赠送礼物的记录
- **SC / Super Chat（绿色左边框）** — 付费醒目留言

**控制按钮**：

- **自动滚动** — 开启后新弹幕自动滚动到可见区域
- **暂停 / 继续** — 暂停或恢复弹幕流的显示刷新
- **清空** — 清空当前弹幕显示记录


## 六、B站读工具

读取 B站 公开数据，无需写权限，可安全调用。填写上方「关键词 / BV号 / UID / 收藏夹ID」等字段，再点击对应按钮：

- **搜索视频** — 按关键词搜索视频。必填：关键词
- **热门视频** — 全站热门视频列表。无必填参数
- **热搜词** — B站 实时热搜榜单。无必填参数
- **每周必看** — 每周必看精选。无必填参数
- **排行榜** — 指定分区排行榜。必填：排序/分区（如 `all` / `game` / `dance`）
- **视频信息** — 获取视频详情。必填：BV号
- **视频评论** — 获取视频评论列表。必填：BV号
- **视频字幕** — 获取 AI 字幕内容。必填：BV号
- **历史弹幕** — 获取视频历史弹幕。必填：BV号
- **用户信息** — 获取用户资料。必填：UID
- **用户投稿** — 获取用户投稿视频列表。必填：UID
- **收藏夹列表** — 获取用户收藏夹列表。必填：UID
- **收藏夹内容** — 获取收藏夹内视频列表。必填：收藏夹 media_id

调用结果统一显示在「B站工具结果」区域。


## 七、B站写工具

对 B站 执行写操作，**会对账号产生影响**，请谨慎使用：

- **发评论/回复** — 在视频下发评论或回复评论。必填：BV号 + 评论内容；回复需额外填写评论 rpid
- **发动态** — 发布新动态。必填：动态文本内容（可附图片）
- **发私信** — 向指定用户发私信。必填：对方 UID + 私信内容

- 开启「是否让NEKO说」后，评论/动态/私信会先交给 NEKO 按人设生成内容，再发送
- 写工具按钮为红色，调用前请确认：账号已登录、内容无误、对象正确


## 八、背景 LLM 设置

启用后，弹幕会被聚合后送给指定 LLM 生成引导词，让 NEKO 更自然地响应直播间氛围。

**基础配置**：

- **启用开关** — 打开/关闭背景 LLM 功能
- **API 地址** — 兼容 OpenAI 格式的端点，如 `https://api.openai.com/v1/chat/completions`
- **模型名称** — 如 `gpt-4o-mini`、`deepseek-chat`
- **API Key** — API 密钥（输入时默认隐藏，可点击查看）
- **聚合窗口** — 收集多少条弹幕后触发一次 LLM 总结，建议 10~20
- **最大采样数** — 弹幕采样池最大容量，超出后按时间淘汰旧弹幕

**进阶设置**（点击「进阶设置」展开）：

- **猫娘名字** — 用于提示词中的 `{name}` 占位符自动替换
- **知识库上下文** — 角色人设、口头禅、常见梗，支持 `{name}` 占位符
- **用户画像摘要** — 主播/用户基本画像，供 LLM 参考
- **提示词模板** — 自定义 System Prompt，支持 `{name}` 和 `{knowledge_context}` 占位符，留空使用默认模板

> 配置完成后点击「保存配置」，然后打开启用开关。点击「测试」可验证 API 连通性。


## 常见问题

**扫码登录失败？** 确认 App 已登录，二维码 2 分钟内有效，刷新后重新扫描。

**弹幕监听无反应？** 检查房间 ID 是否正确、网络是否正常、账号是否已登录。

**AI 不回复弹幕？** 确认推送间隔已设置、背景 LLM 已启用且 API 配置正确。

**发送弹幕失败？** 确认已登录且直播间有发弹幕权限（部分房间限制账号等级）。

**API 调用报错？** 检查 API 地址、模型名称、API Key 是否正确，点击「测试」排查。
